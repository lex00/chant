# Add --continue flag to chant merge for resuming after manual conflict resolution

## Problem

When `chant merge --rebase` encountered a conflict without `--auto`, it would bail out but leave the git repo in a conflicted rebase state. This made it impossible to continue or required manual `git rebase --abort`.

## Solution

Added `git::rebase_abort()` call when conflicts are detected without `--auto`. This ensures the repo is left in a clean state.

## Note

A full `--continue` workflow (storing remaining specs, resuming after manual resolution) was considered but deferred due to complexity. The current fix ensures users aren't left with a broken repo state.

For now, the recommended workflow is:
1. Run `chant merge --rebase --auto` for automatic conflict resolution
2. If `--auto` fails or isn't used, the rebase is aborted and the spec is skipped
3. User can retry individual specs or use `--continue-on-error` to process remaining specs
