# Comprehensive documentation audit - remove unimplemented features

## Problem

Front-facing user documentation currently describes features that are not yet implemented. This creates confusion for users who expect these features to work. According to FEATURE_STATUS.md, several features are marked as "Planned but Not Implemented ❌" but may still be described as available in user-facing docs.

**Principle**: Only the roadmap should reference unimplemented features. All other documentation should describe what exists today.

## Audit Strategy

1. **Read each document** in the priority order above
2. **Search for mentions** of unimplemented features/commands
3. **Evaluate each mention**:
   - Is it describing the feature as available now? → Remove or mark as unimplemented
   - Is it in a "future" or "planned" section? → Move reference to roadmap.md
   - Is it in roadmap.md? → Keep as-is
4. **Update accordingly**:
   - Add "Status: Not Implemented ❌" banners at top of unimplemented feature docs
   - Remove how-to instructions for unavailable features
   - Update command lists to exclude unimplemented commands
   - Add "See roadmap for planned features" links where appropriate
