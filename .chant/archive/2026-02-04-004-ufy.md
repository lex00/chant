# Make chant init --silent perform full initialization silently

## Problem

`--silent` should mean "perform initialization without output", not "skip and be quiet". Currently when already initialized, `chant init --silent` just returns early and does nothing.

In `src/main.rs` lines 1748-1913, the `already_initialized && !force_overwrite` block returns early for silent mode without performing any init actions.

## Solution

When `--silent` is set, skip the "already initialized" early return and proceed with initialization logic. The `--silent` flag should:
1. Perform full init (create dirs, write config, write prompts) if not exists
2. Ensure all components exist if partially initialized
3. Do all of this without any println! output

Modify the early-return logic around line 1909: instead of returning early when `silent`, continue to the actual initialization code but suppress output.

## Acceptance Criteria

- [ ] `chant init --silent` on fresh project creates `.chant/`, config, specs/, prompts/ silently
- [ ] `chant init --silent` on already-initialized project ensures all dirs exist (idempotent)
- [ ] `chant init --silent` produces zero stdout output
- [ ] `chant init --silent` exits 0 on success
- [ ] `chant init` (without --silent) behavior unchanged
