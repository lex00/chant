# Implement agent config separation (no migration needed)

## Problem

.chant/config.md mixes project settings (should be in git) with agent configs (should not be in git). Since chant has no external users yet, we can fix this properly without migration complexity.

## Solution

**Simple fix:**

1. **Remove agents from .chant/config.md**
   - Move parallel.agents section to global config only
   - Update .chant/config.md to remove agents
   - Add agents.md to .chant/.gitignore

2. **Use global config for agents**
   - ~/.config/chant/config.md contains parallel.agents
   - Fix Config::load() to actually use load_merged() everywhere
   - Optional .chant/agents.md for project overrides (gitignored)

3. **Update init command**
   - Don't include parallel.agents in .chant/config.md template
   - Add agents.md to .gitignore
   - Show message: "Configure agents in ~/.config/chant/config.md"

4. **Update this repo's config**
   - Move our agents config to global
   - Remove from .chant/config.md
   - Commit the clean version

## No Migration Needed

- No external users

- No migration commands
- Just ship it correctly
