# Add chant log command to tail spec log file

Add a `chant log` command that tails the log file for a spec.

## Acceptance Criteria

- [x] Add `chant log <id>` subcommand
- [x] Tails `.chant/logs/{spec-id}.log` with follow mode
- [x] If log doesn't exist, show helpful message
- [x] Support `--lines N` to show last N lines (default: 50)
- [x] Support `-f` or `--follow` for continuous tail
- [x] Update CLI docs with examples

## Example Usage

```bash
# Show last 50 lines of spec log
chant log 001

# Follow log in real-time (useful during chant work)
chant log 001 -f

# Show last 100 lines
chant log 001 --lines 100
```

## Implementation Notes

- Use `std::process::Command` to spawn `tail`
- Or implement simple tail in Rust for portability
- Log path: `.chant/logs/{resolved-spec-id}.log`
- Handle partial spec ID resolution like other commands
