# Fix git_user pattern syntax documentation

## Problem

Documentation shows incorrect pattern syntax for `git_user` derivation source. The example uses a regex pattern `".*"` but the implementation only accepts literal strings `"name"` or `"email"`.

**Documented** (docs/enterprise/enterprise.md:628):
```yaml
author_email:
  from: git_user
  pattern: ".*"              # âŒ WRONG - will not work
```

**Implementation Reality** (src/derivation.rs:186-199):
```rust
DerivationSource::GitUser => {
    match field_type {
        "name" => context.git_user_name.clone(),
        "email" => context.git_user_email.clone(),
        _ => None,  // Any other pattern returns None
    }
}
```

The pattern must be the literal string "name" or "email", NOT a regex pattern.

## Solution

Update all documentation examples to use correct git_user pattern syntax.

**Correct Examples:**
```yaml
author_name:
  from: git_user
  pattern: "name"    # Extracts git config user.name

author_email:
  from: git_user
  pattern: "email"   # Extracts git config user.email
```

Add a note explaining that git_user source doesn't use regex - pattern must be literal "name" or "email".
