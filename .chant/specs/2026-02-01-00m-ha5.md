# Make bundled standard.md prompt language-agnostic

The shipped `prompts/standard.md` contains Rust-specific instructions (`cargo fmt`, `cargo clippy`, `just test`, `./target/debug/chant`) that confuse agents working on non-Rust projects.

## Problem

Lines with Rust-specific content:
- Line 21: `cargo fmt`
- Line 22: `cargo clippy`
- Line 23: `just test`
- Line 32, 39, 50-52, 58: `just chant` references
- Line 77: `"just chant" or "./target/debug/chant"`
- Line 81: `"Always ensure chant binary builds"`

## Tasks

1. Update `prompts/standard.md` to use generic language:
   - "Format code according to project conventions" instead of `cargo fmt`
   - "Run linter and fix warnings" instead of `cargo clippy`
   - "Run tests" instead of `just test`
   - "chant list" instead of `just chant list`
   - "chant add" instead of `just chant add`
   - Remove "ensure chant binary builds" constraint
2. Keep the same structure and flow
3. Ensure prompt still works for Rust projects (generic instructions apply)

## Acceptance Criteria

- [ ] No `cargo`, `clippy`, `rustfmt` references in prompts/standard.md
- [ ] No `just` command references in prompts/standard.md
- [ ] No chant-binary-specific references (./target/debug/chant)
- [ ] Instructions are language-agnostic but still actionable
- [ ] Prompt compiles into binary (cargo build succeeds)
