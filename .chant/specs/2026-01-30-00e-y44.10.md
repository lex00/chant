# Integrate Scoring into chant add

Add quality feedback after spec creation in `chant add`.

### Acceptance Criteria

- [ ] Calculate quality score after spec is created
- [ ] Display traffic light, key dimension grades, and suggestions
- [ ] Format: "Quality: {traffic_light}\n  Complexity: {grade} ({details})\n  Confidence: {grade}\n\nSuggestion: {text}"
- [ ] Only show suggestions if status is Review or Refine
- [ ] Ensure spec creation workflow is not disrupted

### Edge Cases

- Spec creation failure should not attempt to score
- Specs with default/empty criteria should show low AC quality

### Example Test Cases

- Creating spec with 3 criteria, 2 files, good structure → shows Ready status
- Creating spec with 10 criteria → shows Refine status with complexity warning
- Creating spec with vague language → shows Review status with confidence suggestion

- Calls scoring functions from Members 2-8
