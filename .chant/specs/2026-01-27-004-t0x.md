# Make bootstrap prompt the default, update docs, prepare release

## Problem

The bootstrap prompt reduces API concurrency issues by starting with a minimal prompt that fetches spec content via `chant prep`. This should be the default behavior.

## Solution

1. Change default prompt from "standard" to "bootstrap" in config.rs
2. Update documentation to reflect bootstrap as default
3. Document the change in CHANGELOG.md for v0.3.2 release
