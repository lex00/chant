# Document all list command flags

The `list` command has many flags that are minimally documented. Users need to understand flag combinations.

## Tasks

1. Read list command definition in src/main.rs
2. Document all flags in `docs/reference/cli.md`:
   - `--ready` (filter to ready specs)
   - `--status` (filter by status)
   - `--global` (search all repos)
   - `--repo` (specific repo)
   - `--project` (project filter)
   - `--json` (JSON output)
   - `--limit` (max results)
3. Document common flag combinations
4. Add examples for each use case

## Acceptance Criteria

- [ ] All list flags documented with descriptions
- [ ] Flag combination examples (e.g., `--global --status in_progress`)
- [ ] Example output shown for `--json` mode
- [ ] Clear explanation of `--global` vs `--repo` vs `--project`
