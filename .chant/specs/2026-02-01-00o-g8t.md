# Separate dev-only prompts from distribution prompts

Dev-only prompts are bundled into the chant binary and shown to all users, but they're only meant for chant development.

## Problem

Current `prompts/` directory mixes:

**General-purpose (for distro):**
- `standard.md` - default agent prompt
- `implement.md`, `research.md`, `review.md` - common workflows
- `split.md`, `triage.md`, `reproduce.md` - utility prompts

**Dev-only (chant development):**
- `doc-audit.md` - auditing chant's own docs
- `documentation.md` - writing chant docs
- `bootstrap.md` - chant bootstrapping
- `research-analysis.md`, `research-synthesis.md` - chant-specific research

All are embedded via `include_str!` in the binary, visible to users via `chant prompt list`.

## Solution Options

1. **Separate directories**: `prompts/` for distro, `prompts-dev/` for dev-only
2. **Prefix convention**: `_doc-audit.md` excluded from bundle
3. **Manifest file**: `prompts/bundle.txt` lists what to include
4. **Build flag**: `--features dev-prompts` includes dev prompts

## Implementation

Option 1 is simplest:
- Move dev prompts to `prompts-dev/` (gitignored from embed)
- Update `build.rs` or embed macro to only include `prompts/`
- Dev prompts still work locally via filesystem lookup
