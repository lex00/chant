# Fix conflict spec template generating invalid YAML - blocked_specs field format

## Problem

The `create_conflict_spec` function in `src/conflict.rs` generates invalid YAML formatting for the `blocked_specs` field in conflict spec frontmatter.

When `blocked_specs` is empty, it generates:
```yaml
blocked_specs:
[]
```

When it has items, it generates:
```yaml
blocked_specs:
- item1
- item2
```

The empty case should be `blocked_specs: []` (on the same line) for consistency with YAML conventions.

## Solution

Modify the template generation in `create_conflict_spec` to:
1. Put `[]` on the same line when empty: `blocked_specs: []`
2. Keep proper list formatting when non-empty:
```yaml
blocked_specs:
  - item1
  - item2
```
