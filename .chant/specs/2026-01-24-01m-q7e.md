# Author CLAUDE.md file with chant workflow constraints

Create the project's CLAUDE.md file documenting how AI agents should work with this codebase.

## Key Constraints to Document

1. **Always use `just chant` for the chant CLI binary** - not `./target/debug/chant` or `cargo run`

2. **Never touch the disk directly** - only the chant CLI gets to write files during spec work

3. **Always use a spec for even the smallest operations** - no ad-hoc changes

## CLAUDE.md Content Should Include

### Project Overview
- Chant is an Intent Driven Development tool
- Specs define work, agents execute specs

### Workflow Rules
- Use `just chant add "description"` to create specs
- Use `just chant work <spec-id>` to execute specs
- Use `just chant list` to see available specs
- Use `just chant show <spec-id>` to view spec details

### Development Commands
- `just chant` - the primary CLI entry point
- Build: `cargo build`
- Test: `cargo test`
- Lint: `cargo clippy`

### File Structure
- `.chant/specs/` - spec files
- `src/main.rs` - main entry point
- `src/spec.rs` - spec parsing/handling

### Important Patterns
- Spec IDs follow format: `YYYY-MM-DD-XXX-abc`
- Split specs use `.N` suffix (e.g., `01e-o0l.1`)
- Driver specs have `type: group`
